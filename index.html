Implementation Playbook

Launch a reliable ops cockpit in three steps.

1. Model your flows

Define plants, carriers, and SKUs in `src/data/`. Use JSON or YAML—just keep relations normalized.

2. Wire automations

Hook your ETL scripts into `scripts/` so nightly refreshes deploy with the site.

3. Share insights

Publish dashboards via GitHub Pages so analysts, schedulers, and execs view the same source of truth.

Contribute

Ideas, fixes, and automation welcome.

Fork the repo, branch from `main`, and follow the guidelines in AGENTS.md. Lint, test, then open a pull request with screenshots or terminal output.

  • Document any new environment variables in `.env.example`.
  • Add fixtures for new data shapers under `tests/fixtures/`.
  • Link an issue so context remains searchable.

Need inspiration?

Check open issues labeled help wanted or submit a proposal in `docs/roadmap.md`.
